# PARTNERS

## Selection rule
Only include partners that improve:
1. demo clarity
2. prize eligibility
3. trust story
4. technical legitimacy

## Current preferred stack
### Must-have
- Base
- Uniswap
- ENS

### Strong candidates
- Self
- Ampersand
- OnchainKit / Base Account
- x402

## Notes
### Base
Home chain, ecosystem alignment, wallet/account UX, strongest event fit.

Critical fit for Hashapp:
- smart wallets
- session keys
- paymaster constraints
- fleet-friendly execution model

### Uniswap
Best concrete value-movement action for the demo and strongest Track 1 partner.

### ENS
Improves readability and trust. Human-friendly payees and agents instead of raw addresses.

### Self
Only worth foregrounding if we land a real privacy or selective disclosure moment.

### Ampersand
Interesting if it becomes the request / approval communication backbone between app and agent.

### x402
Only worth featuring if directly in the payment loop for paid agent services.

### OnchainKit / Base Account
Strong implementation partner for polished wallet UX, identity continuity, and agent-friendly wallet setup.

## Anti-pattern
Do not create a sponsor salad. The stack should feel inevitable, not stitched together.
