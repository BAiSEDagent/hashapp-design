My recommendation
Make the toggle real
It should control:
privateReasoningEnabled for the linked agent
persisted in state
reflected in UI copy

At minimum:
On → Venice/private review available
Off → Venice/private review disabled, provenance hidden for future actions

Do not make the whole card disappear
That gets jumpy and dumb.
Better:
keep the card visible
use the switch for state
use an accordion/disclosure section for detail

Best interaction model
Card header
title: Reasoning & Privacy
real On/Off switch
short status text

Expandable body
Collapsed by default if you want cleaner UI.
Expanded state shows:
provider: Venice
disclosure policy
input types
last analysis
short explanation

That is better than a hard show/hide.

Why
Because the product needs to communicate:
privacy posture
not just hide settings when off

A dead switch is bad.
A disappearing card is also bad.
A live switch + accordion is the right middle ground.

Suggested behavior
ON
switch active
body can expand
status says:
Private review enabled
provenance can appear on future actions

OFF
switch inactive
body still expandable
status says:
Private review disabled
explanatory copy:
“Actions will not use Venice private reasoning.”
