What Hashapp is
Hashapp is a consumer-grade spending app for AI agents.
It should feel like a premium, mobile-first finance app where a human can safely let an agent buy what it needs.

It should not feel like:
a crypto dashboard
a terminal UI
an admin panel
a wallet explorer

Core demo lane
Use the research agent scenario.

Named agent: Scout

The hero of the product is the activity feed.

Required v1 surfaces
Activity feed
Receipt detail
Approval request flow
Rules / preset screen
Agent detail
Payees / trusted destinations

Required demo behavior
Open on the Activity screen
Show recent purchases from Scout
A new purchase request appears live
User approves it
Receipt appears in the feed
User changes one plain-English rule
Scout attempts another purchase
The request is blocked
Blocked state appears clearly in the feed

Product rules
show intent, not raw tx data
plain-English rules only
human-readable failure states
avatar-first identity
one clear next action per state
the feed is the product

Anti-patterns
Do not build:
a cyber dashboard
a terminal UI
chart-heavy admin software
raw-hex-first wallet screens
multiple competing primary CTAs

Success condition
A judge should see the prototype for 30 seconds and immediately understand:
this is the spending app for AI agents
this makes agent purchases feel safe
this is consumer-grade
this should exist

Please build a strong first pass of the product surface with these constraints and prioritize the quality of the activity feed above everything else.
