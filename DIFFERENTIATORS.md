# Differentiators

Hashapp’s advantage is not just scoped spending. It is making agent spending feel safe, legible, and useful to normal humans.

## 1. Emergency Freeze
A visible kill switch to pause all agent spending, revoke active permissions, or freeze recurring permissions instantly.

## 2. Permission Presets as Personalities
Permission bundles expressed as understandable behaviors:
- Intern
- Researcher
- Trader
- Ops
- Manual Only

## 3. Pre-Approved Trusted Destinations
Allow users to proactively add trusted destinations before the agent asks.

## 4. Why Approved / Why Blocked Layer
Every action should have a plain-English explanation for why it was allowed or denied.

## 5. Permission Timeline
A visible history of permission creation, changes, revocations, expiries, and usage.

## 6. Smart Suggestions
Suggestions like:
- add vendor to trusted list
- create recurring permission
- lower or raise a budget based on behavior

## 7. Private Review Toggle
An optional private reasoning layer for unusual or sensitive requests.

## 8. Human / Agent / Vendor Identity Layer
Readable relationship model for humans, agents, and trusted destinations using names, avatars, and identity surfaces.

## 9. Auto-Approve Confidence Ladder
More nuanced autonomy modes beyond simple on/off approval.

## 10. Receipt as a First-Class Object
Receipts become reusable proof objects, not just transaction detail screens.
